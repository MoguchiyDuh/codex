# Interfaces & Abstract Classes

> Tools for defining contracts — what something does, not how.

## Interface

### Pure contract: no implementation, no state

### Enables polymorphism without inheritance

### Multiple interface implementation

## Abstract class

### Partial implementation — mix of concrete and abstract methods

### Can have state (fields)

### Single inheritance (in most languages)

## Interface vs abstract class — when to use each

| | Interface | Abstract Class |
|---|---|---|
| State | No | Yes |
| Multiple | Yes | No (usually) |
| Use when | Defining capability | Sharing partial implementation |
